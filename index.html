Intro to Git & GitHub
Why should you learn this?
  • +To build good development habits ðŸ˜Ž
  • +In the long term it will save you time and energy
  • +It's just as important as saving your Adobe files!
  • +You can contribute to open source communities (e.g., Processing)
Setup (The boring stuff)
  • +Create a GitHub account if you don't already have one
  • +Install iTerm (unless you want to use the built in Terminal app)
  • +Git should be installed by default, but verify in your terminal app: git --version
  • +You should have a text editor installed, like Atom, Sublime, or Visual Studio Code
What is Git?
  • +Git is a Version Control System (VCS) that tracks file changes in a directory
  • +It allows you to essentially 'backup' your code!
  • +You can create different versions of your code, helpful for experimenting + feature building
  • +You can think of git as a set of commands that do the above
  • +We can call these commands using the Git CLI (Command Line Interface)
What is GitHub?
  • +GitHub is a hosting service for your code repositories
  • +GitHub comes with features, e.g., GitHub pages
  • +There are other hosting services out there (Bitbucket, GitLab)

Part II: Cloning your first repo
Some terminology
  • +We will start by 'cloning' a remote repository (repo)
  • +Remote: an existing repository that is hosted somewhere (in this case, on GitHub)
  • +Cloning: making a copy of it locally (downloaded to your machine)
Cloning a repo
  • +cd into the directory where you want the repo to be cloned, e.g., Documents or Desktop
  • +link to repo here 
  •                   
                    # Cloning this repo to my Documents directory
                    Rachels-MacBook-Pro:~ rosewater$ cd Documents
                    Rachels-MacBook-Pro:Documents rosewater$ git clone https://github.com/rose-water/boilerplate-html.git
Open the code in your text editor
